# 에이전트 프롬프트 예시

아래 프롬프트 예시를 참고해서 보다 에이전트를 활용하시기 바랍니다!

#### 💡 복합 프롬프트 예시
```
"[사용자 관리 기능]을 구현해야 해.
1. 먼저 PDCA plan으로 설계를 작성하고,
2. TDD 방식으로 API를 먼저 개발한 후,
3. Phase-6 UI 통합으로 프론트엔드를 연동해줘.
4. 완료 후 Phase-8 갭 분석으로 누락된 부분 확인해줘."
```

```
"이 에러로그를 분석해줘:
1. Systematic Debugging으로 근본 원인 파악
2. 관련 코드를 Code Review해서 취약점 확인
3. TDD로 재현 테스트 작성 후 수정
4. PDCA report로 해결 보고서 생성"
```

#### 커맨드

#### 🎯 핵심 개발 스킬
| 스킬 | 프롬프트 예시 |
|------|-------------|
| **PDCA (설계)** | "PDCA 스킬로 [기능명]에 대한 구현 계획서를 먼저 작성해줘." |
| **PDCA (분석)** | "`/pdca analyze [기능명]`으로 설계와 구현 간 갭 분석을 진행해줘." |
| **PDCA (보고)** | "작업 완료 후 `/pdca report [기능명]`으로 최종 보고서를 생성해줘." |
| **Systematic Debugging** | "Systematic Debugging 스킬을 사용하여 [에러 내용]의 근본 원인을 분석하고 해결책을 제시해줘." |
| **TDD** | "TDD 프로세스에 따라 [함수/모듈]을 개발해줘. 테스트 코드를 먼저 작성해야 해." |
| **Code Review** | "이 파일의 코드를 [성능/보안/유지보수] 관점에서 심층 리뷰해줘." |
| **Subagent-driven** | "복잡한 구현 태스크이므로 Subagent-driven-development 스킬을 사용하여 작업을 분할하고 처리해줘." |
| **Writing Plans** | "이 요구사항을 2-5분짜리 bite-sized 태스크로 분해해서 계획을 작성해줘." |

#### 🔬 개발 단계별 스킬 (Phase 1~9)
| 스킬 | 프롬프트 예시 |
|------|-------------|
| **Phase-1 (스키마)** | "프로젝트에서 사용할 용어와 데이터 모델을 정의해줘. (phase-1-schema)" |
| **Phase-2 (컨벤션)** | "이 프로젝트의 코딩 컨벤션과 네이밍 규칙을 정리해줘. (phase-2-convention)" |
| **Phase-3 (목업)** | "디자이너 없이 [화면명]의 HTML/CSS 프로토타입 목업을 만들어줘. (phase-3-mockup)" |
| **Phase-4 (API)** | "[기능명]에 필요한 REST API 엔드포인트를 설계해줘. (phase-4-api)" |
| **Phase-5 (디자인 시스템)** | "프로젝트에서 재사용할 컴포넌트 라이브러리와 디자인 토큰을 정의해줘. (phase-5-design-system)" |
| **Phase-6 (UI 통합)** | "프론트엔드와 백엔드 API를 연동해줘. 상태 관리 패턴도 적용해. (phase-6-ui-integration)" |
| **Phase-7 (SEO/보안)** | "이 페이지의 SEO 메타 태그와 보안 취약점을 점검해줘. (phase-7-seo-security)" |
| **Phase-8 (리뷰)** | "설계서와 현재 코드 간의 갭 분석을 수행해줘. (phase-8-review)" |
| **Phase-9 (배포)** | "CI/CD 파이프라인과 프로덕션 배포 설정을 해줘. (phase-9-deployment)" |
